# テスト管理システム 画面ワイヤーフレーム（MVP）

## 1. 画面一覧
1. リリース一覧画面  
2. テストケース一覧画面（メイン）  
3. テストケース詳細画面  
4. テスト実行入力（モーダル or 画面）

---

## 2. リリース一覧画面

### 目的
- リリース単位で進捗状況を即座に把握する
- 未完了が多いリリースを早期に認識する

### ワイヤーフレーム（ASCII）

    [ テスト管理システム ]

    -----------------------------------------
    | リリース一覧                         |
    -----------------------------------------
    | [+ 新規リリース]                     |
    -----------------------------------------
    | リリース名 | 期間 | 進捗 | 未完了 | 更新 |
    -----------------------------------------
    | R2025-01   | 4/1- | 75%  | 5件    | 6/10 |
    | R2025-02   | 5/1- | 20%  | 24件   | 6/12 |
    -----------------------------------------

### 表示項目
- リリース名（クリックでテストケース一覧へ遷移）
- 期間（開始日〜終了日）
- 進捗率（合格数 ÷ 対象件数）
- 未完了件数
- 最終更新日時

---

## 3. テストケース一覧画面（メイン画面）

### 目的
- 日常的なテスト作業の中心画面
- 今やるべきテストとリスクを即座に判断できる

### 上部コンテキスト・操作エリア

    リリース：R2025-01
    進捗：75%（Pass 30 / 対象 40）

    [ + テストケース追加 ]

    [ ステータス▼ ] [ 担当者▼ ] [ 未完了のみ ☑ ] [ リスクのみ ☑ ]

### 一覧ワイヤーフレーム

    --------------------------------------------------------------------------------
    | 画面名        | 状態     | 担当 | 最終結果 | 実行日時 | 更新日時 |
    --------------------------------------------------------------------------------
    | ログイン画面  | 合格     | 山田 | Pass     | 6/10     | 6/10     |
    | 会員登録画面  | 不合格   | 佐藤 | Fail     | 6/11     | 6/11     |
    | 設定画面      | 未着手   | 田中 | -        | -        | 6/09     |
    | 注文画面      | ブロック | 山田 | Blocked  | 6/12     | 6/12     |
    --------------------------------------------------------------------------------

### 表示ルール（視認性）
- 合格：🟢
- 未着手 / 実施中：⚪
- 不合格：🔴
- ブロック：🟠
- 保留：🟡

---

## 4. テストケース詳細画面

### 目的
- 対象画面の状態・履歴・鮮度を1画面で把握する

### ワイヤーフレーム

    -----------------------------------------
    画面名：ログイン画面
    -----------------------------------------
    URL：/login
    担当者：山田
    状態：不合格 🔴
    優先度：High

    最終実行：
      結果：Fail
      実行者：佐藤
      実行日時：2025/06/11 14:30

    -----------------------------------------
    ■ 確認観点
    - 正常ログインできる
    - エラーメッセージ表示
    - ロック処理

    ■ 期待結果
    - 正しいID/PWでログイン成功
    - 誤入力でエラー表示

    -----------------------------------------
    ■ 実行履歴
    -----------------------------------------
    | 日時 | 結果 | 実行者 | コメント |
    -----------------------------------------
    | 6/11 | Fail | 佐藤 | エラー文言違い |
    | 6/10 | Pass | 山田 | 初回OK |
    -----------------------------------------

    [ テスト実行する ]

### 必須表示項目
- 作成日時 / 更新日時
- 作成者 / 更新者
- 実行履歴（時系列）

---

## 5. テスト実行入力（モーダル）

### 目的
- 入力を迷わせず、情報不足を防ぐ

### ワイヤーフレーム

    -------------------------------
    テスト実行
    -------------------------------
    結果：
    ( ) Pass
    ( ) Fail
    ( ) Blocked
    ( ) 保留

    コメント：
    [_________________________]

    証跡URL：
    [_________________________]

    [ 実行結果を保存 ]
    -------------------------------

### 入力ルール
- Pass：コメント任意
- Fail / Blocked / 保留：コメント必須
- 保存時に以下を更新する
  - テスト実行履歴（test_runs）を追加
  - テストケースの status / last_result / last_executed_at を更新

---

## 6. 設計方針（ドメイン観点）
- 管理より「判断」を優先するUI
- 更新日時が常に見える構造
- 未完了・リスクが即座に分かる色分け

この設計により  
「いつの情報かわからない」「どこが危険かわからない」問題を構造的に解消する。
